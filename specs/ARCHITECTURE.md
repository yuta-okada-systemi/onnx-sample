## データフロー

```
1. ユーザー入力（リポジトリ, トークン）
         ▼
2. [GitHub API 通信] ──► Issue データ取得
         ▼
3. [IndexedDB 保存] ──► ローカルストレージに格納
         ▼
4. ユーザー検索入力
         ▼
5. [モデルローダー] ──► Transformers モデル DL & 初期化
         ▼
6. [テキスト埋め込み] ──► 検索クエリ & Issue テキストをベクトル化
         ▼
7. [Search Engine] ──► BM25 検索 + Semantic 検索を実行
         ▼
8. [RRF統合] ──► ハイブリッド結果をランキング統合
         ▼
9. [結果表示] ──► UI に結果を表示
```

## コンポーネント説明

### 1. Side Panel (popup.html, popup.js)
**責務**: ユーザーインターフェース
- 検索入力フォーム
- 結果表示
- 設定管理（リポジトリ, GitHub トークン）
- 同期ボタン・エクスポートボタン

### 2. Background Service Worker (background.js)
**責務**: バックグラウンド処理
- GitHub API との通信
- Issue データの同期
- 検索エンジン実行
- IndexedDB 操作

### 3. Search Engine (検索ロジック)
**責務**: ハイブリッド検索実装
- **BM25**: 字句検索（タイトル重み: 3x、本文: 1x）
- **Semantic Search**: トランスフォーマーで埋め込み生成
- **RRF**: 2 つのランキングを k=60 で統合

### 4. Model Loader (Transformers)
**責務**: ML モデル管理
- Hugging Face CDN からモデルダウンロード
- ローカルキャッシュ
- テキスト埋め込み生成

### 5. Storage (IndexedDB)
**責務**: ローカルデータ永続化
- Issue ドキュメント格納
- セマンティック埋め込みキャッシュ
- 字句インデックス

## 検索アルゴリズム

### BM25 (字句検索)
```
スコア = IDF(q) × (k1+1) × f(qi,D) / (k1×(1-b+b×|D|/avgdl) + f(qi,D))

パラメータ:
- k1 = 1.5 (term frequency saturation)
- b = 0.75 (document length normalization)
- タイトル重み: 3x
- 本文重み: 1x
```

### Reciprocal Rank Fusion (RRF)
```
最終スコア = Σ(BM25スコア, セマンティックスコア) 重み付け統合
パラメータ: k = 60

max_lexical_rrf_boost = 0.08 (レキシカルブースト上限)
```

## 技術的な特徴

| 特徴 | 説明 |
|------|------|
| **Manifest V3** | 最新 Chrome 拡張機能標準 |
| **Service Worker** | 拡張機能の永続処理 |
| **WASM** | CSP に `wasm-unsafe-eval` 許可（Transformers WASM実行用） |
| **IndexedDB** | オフライン検索対応 |
| **Semantic + Lexical** | ハイブリッド検索で精度向上 |
| **プライベート処理** | 外部サーバーに検索クエリ未送信 |
